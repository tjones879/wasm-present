What is WASM?

Web assembly serves as a new compilation target for web developers. This standard is being developed by major players including Apple, Google, Microsot, and Mozilla.

Why is it useful?

A brief history of javascript must first be understood. Javascript has historically been powerful enough for most of our needs in the web.

However, large file sizes and poor performance in comparison to native languages create significant problems within certain contexts.

Alternatives

Using Emscripten, you can already compile many source languages (Haskell, etc.) into relatively fast JS.

Developers needed more power, so a special subset of JS called asm.js was declared. This allows for javascript to be executed much closer to native speeds. (2x difference)

Who uses it, and is it stable?

Currently, web assembly is not yet used. There are multiple issues preventing widespread adoption, namely:

  1. The standard is still rapidly changing.
  2. Browser compatibility is lacking.
  3. Source languages are few.

What is a working example?

We must first enable our browser to run wasm. You must be running at least Firefox 52 or Chrome 57 and enable special flags, just as with WebGL.

Example in Chrome:
chrome://flags/#enable-webassembly
